# AvitoTech ML CUP 2026: Recommendation System

Репозиторий команды по разработке системы ранжирования для AvitoTech ML CUP.

## 🛠 Окружение и настройка

Мы используем **Python 3.12** (рекомендуется любая версия **3.10+** для корректной работы Polars и CatBoost), **WSL (Ubuntu)** и пакетный менеджер **uv** для быстрой установки зависимостей.

### 1. Склонируйте репозиторий

```bash
git clone git@github.com:ваш_логин/avito_recsys.git
cd avito_recsys
```

### 2. Создайте и активируйте виртуальное окружение

```bash
python3 -m venv venv
source venv/bin/activate
```

### 3. Установите зависимости

Рекомендуется использовать [uv](https://github.com/astral-sh/uv) для ускорения загрузки тяжёлых библиотек (CatBoost, Polars). Если `uv` ещё не установлен:

```bash
# Установка uv
curl -LsSf https://astral.sh/uv/install.sh | sh
# Добавьте uv в PATH (или перезапустите терминал)
export PATH="$HOME/.local/bin:$PATH"

# Установка пакетов
uv pip install -r requirements.txt
```

### 4. Настройте Git-фильтры (критично!)

Чтобы не плодить конфликты в Jupyter-ноутбуках и не пушить тяжёлые аутпуты, обязательно инициализируйте `nbstripout`. Убедитесь, что он установлен (входит в `requirements.txt` или установите вручную: `pip install nbstripout`):

```bash
nbstripout --install
```

## 📂 Структура проекта

- `data/` — все данные (полностью в `.gitignore`):  
  `raw/` — сырые файлы, `processed/` — обработанные датасеты.
- `notebooks/` — песочницы для EDA и экспериментов. Все ноутбуки проходят очистку перед коммитом.
- `src/` — общий production-код: загрузчики данных, кастомные метрики, утилиты.
- `models/` — сохранённые веса моделей и логи обучения (будет добавлена позже).
- `smoke_test.py` — быстрый тест целостности окружения и данных.

---

## 🚦 Правила работы (Workflow)

1. **Данные:** никогда не добавляйте файлы из `data/` в индекс git. Если создали полезный сабсет — опишите способ его получения в коде.
2. **Ноутбуки:** перед коммитом убедитесь, что `nbstripout` работает. В репозиторий должен попадать только код и разметка, без результатов выполнения ячеек.
3. **Зависимости:** при установке новой библиотеки обязательно обновите конфиг:
   ```bash
   uv pip freeze > requirements.txt
   ```
4. **Код:** если какая-то функция в ноутбуке полезна всем (например, расчёт NDCG), выносите её в `src/utils.py`.
