# Experiments Log

## Baseline pipeline: ALS retrieval + CatBoost reranking

**Дата:** 7/8/2026

**Методология:** 

synth-eval через `prepare_local_eval.py` (threshold 2026-04-08),
val-сплит по юзерам (20%, 1426 юзеров с таргетом из local_eval.csv).

**Известное ограничение:** 

ALS обучен на полном train до 2026-04-15, то есть "видел"
события synth-eval окна (2026-04-08 $\rightarrow$ 2026-04-15) — лёгкая оптимистичная утечка.
Метрики сопоставимы между собой (одинаковая утечка для всех), но не строго
сопоставимы с реальным лидербордом соревнования — доступа к оригинальному тесту нет.

| Метод              | Recall@160 | Комментарий |
|--------------------|-----------|-------------|
| Popular baseline   | 0.0078    | top-160 популярных item в 6 разрешённых вертикалях |
| ALS retrieval      | 0.0368    | top-1 по als_score, N=400 кандидатов на юзера |
| CatBoost ranker    | 0.0416    | YetiRank, переранжирование топ-400 ALS-кандидатов |

**Retrieval recall (потолок для ранкера):** 

7.62% — доля таргет-пар, попавших
в top-400 ALS-кандидатов. CatBoost физически не может превысить эту границу.

**Внешний ориентир:**

top-1 ~ 0.177, top-10 ~ 0,124, top-100 = 0,037.

**Вывод:** 

двухэтапная схема работает как задумано — ALS даёт основной прирост
над популярностью (~x5), CatBoost добавляет ещё +14.2% поверх ALS.
Основное узкое место — retrieval recall (7.62%), а не ранкер. Следующий шаг —
расширение источников кандидатов. 

## Эксперимент 2: + co-visitation как второй источник кандидатов

**Дата:** 7/9/2026

| Метод | Recall@160 | Retrieval recall (потолок) |
|---|---|---|
| Popular baseline | 0.0078 | — |
| ALS retrieval | 0.0368 | 7.62% |
| Covisit retrieval | 0.1364 | 17.46% |
| Combined (ALS $\cup$ covisit) | — | 21.27% |
| CatBoost на merged | **0.1611** | 21.27% |

**Параметры covisit:** 

* 28 дней
* eid $\neq$ 7 
* depth = 3
* gap $\leq$ 24 часа
* симметрично
* top-50 соседей/item
* MIN_PAIR_USERS $\geq$ 3
* 15 затравок
* top-400 кандидатов
Матрица: 2.52М связей, 1.15М item (3.3% каталога).

**Feature importance:** 

* item_n_users (20.4)
* item_popularity (19.4)
* covisit_rank (12.6)
* als_rank (11.5)
* Флаги in_als/in_covisit избыточны (< 0.3).

**Вывод:**

co-visitation — главный спусковой крючок (+271% над ALS). Ранкер даёт +18%
поверх лучшего источника. Узкое место сместилось с retrieval на качество
ранжирования и полноту covisit-матрицы (покрытие всего 3.3% каталога).

**Известное ограничение (утечка через окно обучения):**

ALS и co-visitation матрица построены на данных вплоть до 2026-04-15,
включая 7-дневное окно (04-08 $\rightarrow$ 04-15), которое одновременно служит
источником synth-eval таргетов. Это означает, что retrieval-модели имели
доступ к тем же событиям (или их части), из которых собран таргет —
классическая temporal leakage. Эффект не измерен точно (для этого
потребовался бы честный прогон с cutoff=04-08 без нахлёста), но
качественно ожидается смещение метрик в сторону завышения, примерно
пропорционально доле этого окна в общем объёме обучающих данных (~25%
по времени, хотя не все события окна становятся таргетами). Метрики
в этом эксперименте валидны для СРАВНЕНИЯ методов между собой (все
затронуты одинаково), но не для прямого сопоставления с официальным
лидербордом.
