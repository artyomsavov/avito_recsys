# Прояснение требований
- **Бизнес-цель**: 
	- Максимизировать вероятность контакта пользователя с предложенными объявлениями(оценивается метрикой Recall@160). 
	- Ключевое условие — обеспечить баланс качества рекомендаций по всем 8 категориям каталога, не допустив доминирования одной популярной категории.
- **Функции, которые должна поддерживать система:**
	- Генерация персонализированного списка из 160 уникальных рекомендаций для каждого целевого пользователя.
	- Система должна поддерживать фильтрацию исключения просмотренного (anti-join), чтобы рекомендовать для пользователя исключительно новые объявления.
- **Данные:** 
	- Обезличенный исторический набор контактных событий (кликстрим состоит из событий 11 типов) пользователей платформы. 
	- Обучающая выборка (`train`) содержит около 5 миллиардов событий.
	- Для инференса доступна история 94408 пользователей (`eval_user_events`) и признаки объявлений (`item_futures`).
- **Ограничения:**
	- Формат взаимодействия с системой — отправка CSV-файла (максимум 160 строк на пользователя).
	- Лимит на тестирование: 5 отправок в день.
	- Сроки соревнования: 27 мая 2026 года.
- **Масштаб системы:**
	- Огромный пространство поиска.
	- Предстоит обработать 5 миллиардов событий и отбирать 160 лучших кандидатов из миллионов объявлений для почти 100 тысяч пользователей за один прогон.
- **Производительность:** 
	- Жёстких требований к задержке ответа модели нет (non real-time) или ограничений в пропускной способности. 
	- Единственное требование — обучение и генерация предсказаний должно укладываться в разумные рамки производительности на нашем оборудовании.
        *Из-за масштаба системы **невозможно** использовать тяжелые модели напрямую на всём каталоге. Это требование обязывает нас перейти к многоступенчатой архитектуре (Retrieval -> Filtering -> Ranking), иначе система просто не сойдется по времени и памяти.*
- **Конфиденциальность и этика:**
	- Все исходные идентификаторы пользователей подаются уже анонимизированными.
	- Действия предоставляются в обезличенным формате.
	- Основное этическое требование к алгоритму — алгоритмическая справедливость, гарантирующая равномерное предоставление разных категорий товаров в выдаче. 
		*Примечание*: 94 408 целевых пользователей принудительно сэмплируются в 10 равных корзин (5 строго вертикальных и 5 общих, до 10 000 человек в каждой). Поскольку итоговая метрика Recall@160 считается как среднее по всем пользователям, математически невозможно получить высокий общий скор, если модель игнорирует нишевые вертикали.

Резюмируя, требуется разработать офлайн-систему рекомендаций, которая предскажет топ-160 уникальных объявлений для 94 408 целевых пользователей, с которыми они с наибольшей вероятностью совершат контакт.

Модель должна обучаться на графе из 5 млрд исторических обезличенных событий. Ключевая специфика задачи заключается в строгой фильтрации кандидатов (исключение уже просмотренных объявлений) и необходимости максимизировать метрику Recall@160 таким образом, чтобы качество рекомендаций оставалось стабильно высоким во всех целевых вертикалях каталога, не допуская смещения в сторону наиболее популярных категорий.
